# MoneyTracker - 1Money Style Implementation

## Обзор

Полностью переписанный интерфейс в стиле популярного приложения 1Money с фокусом на визуальную инфографику и интуитивный UX.

## Ключевые особенности

### 1. Главный экран - Дашборд (`/main/transactions/index.tsx`)

**Верхняя панель:**
- **Селектор счета**: Выпадающее меню с названием счета и текущим балансом
- **Баланс**: Отображение общего баланса всех счетов

**Таймлайн (Период):**
- Горизонтальный скролл с пресетами: День, Неделя, Месяц, Год, Период
- Свайп для переключения между периодами (будет реализован)
- Фильтрация транзакций по выбранному периоду

**Центральный элемент - Кольцевая диаграмма (Donut Chart):**
- Большой "бублик" (donut chart) с градиентными сегментами
- Размер сегмента пропорционален доле категории в общих тратах
- **Центр круга**: Крупная сумма с цветовой кодировкой (красный для расходов, зеленый для доходов)
- **Иконки вокруг круга**: Круглые иконки категорий на внешнем радиусе кольца
- Нажатие на иконку показывает транзакции только по этой категории
- Легенда с процентами под диаграммой

**Список операций (Шторка):**
- Группировка транзакций по дате
- Каждая транзакция отображается как карточка:
  - Круглая иконка категории в заданном цвете
  - Название категории (крупно) + заметка (мелко)
  - Указание счета
  - Сумма с цветовой кодировкой (+ зеленый, − красный)

**Кнопка добавления:**
- Floating Action Button (FAB) справа внизу
- При нажатии открывает модальное окно

### 2. Создание транзакции (`/src/components/ui/AddTransactionModal.tsx`)

**Модальное окно (Bottom Sheet):**
- Выезжает снизу с анимацией slide
- Полупрозрачный фон

**Вкладки (Тип операции):**
- Три вкладки: Расход, Доход, Перевод
- Переключатель сверху модального окна
- Цветовая кодировка: красный/зеленый

**Дисплей калькулятора:**
- Огромное поле для ввода суммы
- Поддержка математических операций (+, -, *, /)
- Кнопки: 1-9, точка, 0, удаление
- Кнопка "С" для очистки
- Форматирование суммы (разделители разрядов)

**Панель деталей (под суммой):**
- **Календарь**: Изменить дату транзакции
- **Заметка**: Добавить текстовый комментарий
- **Кошелек**: Выбрать счет для операции

**Выбор категории:**
- Нажатие на "+" открывает модальное окно выбора категории
- Сетка иконок категорий с цветами
- Поиск по категориям
- Превью выбранной категории в модальном окне

**Кнопка подтверждения:**
- Большая кнопка с галочкой
- Цвет зависит от типа транзакции
- Отключена, если не все поля заполнены

### 3. Создание категории (`/main/categories/create.tsx`)

**Верхний блок (Превью):**
- Большой круг превью категории (120x120px)
- Изменяется в реальном времени при настройке
- Отображение иконки, названия и типа

**Настройка внешнего вида:**
- **Палитра цветов**: Горизонтальная прокручиваемая лента
  - 18 ярких контрастных цветов (flat colors)
  - Цвета не сливаются друг с другом на графике
  - Галочка на выбранном цвете

- **Каталог иконок**: Вертикальный скролл с группировкой
  - 8 категорий: Еда, Транспорт, Дом, Покупки, Развлечения, Здоровье, Финансы, Другое
  - Каждая категория содержит 8 иконок
  - Всего 64 иконки для выбора
  - Минималистичный стиль (эмодзи)
  - Выделение выбранной иконки рамкой

**Дополнительные параметры:**
- **Тип операции**: Расход / Доход (переключатель)
- **Бюджетный лимит**: Поле для установки лимита на месяц
- **Скрытие**: Тумблер "Не учитывать в отчетах"

**Фиксированная кнопка создания:**
- Всегда внизу экрана
- Цвет зависит от типа категории
- Отключена, если не все поля заполнены

## Техническая реализация

### Компоненты

**DonutChart** (`/src/components/ui/DonutChart.tsx`):
- SVG-диаграмма с градиентными сегментами
- Использует `react-native-svg` и `react-native-reanimated`
- Pinch-to-zoom жест для масштабирования
- Динамическое позиционирование иконок по окружности
- Легенда с процентами

**AddTransactionModal** (`/src/components/ui/AddTransactionModal.tsx`):
- Bottom sheet модальное окно
- Калькулятор с математическими операциями
- Интеграция с dataStore для создания транзакции
- CategoryPickerModal для выбора категории

### State Management

**Zustand Store**:
- `transactions` - все транзакции пользователя
- `categories` - категории (системные + персональные)
- `accounts` - счета пользователя
- `addTransaction` - метод создания транзакции
- `addCategory` - метод создания категории

### Типизация

Полная поддержка TypeScript:
- `Transaction` с полями: id, userId, accountId, categoryId, amount, type, description, date
- `Category` с полями: id, userId, name, type, icon, color, isBaseNeed
- `TransactionType`: INCOME, EXPENSE, TRANSFER
- `CategoryType`: INCOME, EXPENSE

### Цветовая кодировка

**Расходы:**
- Основной цвет: `#FF3B30` (красный)
- Палитра: красные, оранжевые, желтые оттенки

**Доходы:**
- Основной цвет: `34C759` (зеленый)
- Палитра: зеленые, голубые, синие, фиолетовые оттенки

## Тестирование

### Запуск:

```bash
# Backend
cd backend
yarn start:dev

# Mobile
cd mobile
yarn start
```

### Тестирование сценариев:

1. **Добавление расхода:**
   - Нажмите на FAB (кнопку +)
   - Выберите "Расход"
   - Введите сумму через калькулятор (например: 1000 + 500)
   - Нажмите на кнопку категории, выберите "Продукты"
   - (Опционально) Добавьте заметку
   - Нажмите на галочку для сохранения

2. **Просмотр диаграммы:**
   - Откройте экран транзакций
   - Посмотрите на кольцевую диаграмму
   - Нажмите на иконку категории вокруг диаграммы
   - Список транзакций отфильтруется по этой категории

3. **Создание категории:**
   - Перейдите в настройки > Создать категорию
   - Введите название (например: "Такси")
   - Выберите тип "Расход"
   - Выберите цвет из палитры
   - Выберите иконку из каталога (Транспорт > 🚕)
   - (Опционально) Установите бюджетный лимит
   - Нажмите "Создать категорию"
