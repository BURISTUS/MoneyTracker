# MoneyTracker — Полное описание приложения

## Бизнес-контекст

**MoneyTracker** — мобильное приложение для управления личными финансами с AI-ассистентом. Помогает пользователю осознанно тратить, копить и инвестировать. Основная идея: показать реальную стоимость покупок в часах работы, а не в деньгах — чтобы каждый расход воспринимался через время жизни.

**Стек:** React Native (Expo) + NestJS + PostgreSQL + Prisma + Redis

**Мобильное приложение:** Expo Router (file-based routing), Zustand (state), i18next (5 языков: en, ru, es, de, pt), Thema (light/dark)

**Бэкенд:** NestJS, Prisma ORM, PostgreSQL на порту 5433, JWT-авторизация, DeepSeek API для AI

## Фичи — детальное описание

### 1. Главная страница (Home)

**Обзор (вкладка Overview):**
- Герой-блок: «Твоя жизнь стоит X ₽/час» — показывает часовую ставку пользователя и баланс за месяц
- Статистика: количество транзакций, баланс, самая крупная категория расходов
- Сводка за месяц: доходы, расходы, экономия (в рублях и в рабочих часах)
- Пульс дня: мини-список сегодняшних транзакций
- Быстрые действия (5 кнопок): Расход | Голос | Инкубатор | Чек | Доход

**Чтение (вкладка Read):**
- Список статей с бэкенда, с переводами на язык пользователя
- Карточки: заголовок, описание, тег, время чтения, счётчик просмотров
- Клик ведёт на экран Articles/[id] с полным текстом

### 2. Транзакции

**Список:**
- Фильтрация по дате, типу (доход/расход/перевод), категории
- Группировка по дням
- Сумма отображается в валюте пользователя с форматированием

**Создание транзакции (3 способа):**
- **Ручной ввод:** выбор типа, суммы, категории, счёта, описания, даты
- **Голосовой ввод:** нажатие на микрофон → распознавание речи → отправка текста на DeepSeek → парсинг в структурированную транзакцию (сумма, категория, описание)
- **Фото чека:** выбор фото → отправка base64 на DeepSeek Vision → распознавание позиций чека → создание одной или нескольких транзакций

**Инкубатор (Wishlist):**
- Пользователь добавляет желаемую покупку: название, цена, описание
- Статус: PENDING (7 дней ожидания) → READY (можно решить) → REJECTED / PURCHASED
- Показывает стоимость в часах работы пользователя
- При отклонении — ачивка «Отказ от импульсивной покупки»

### 3. Чат с AI-ассистентом

- 4 пресета: Отчёт по тратам | Анализ бюджета | Как сохранить | Динамика
- Свободный ввод текста
- Сохранение истории в БД (ChatMessage: userId, role, content, presetType)
- Ответы форматируются: нумерованные списки, жирный текст, буллеты
- Индикатор набора (анимация точек)
- Возможность очистить историю

### 4. Вишлист (Wishlist)

- Список желаний со статусами (pending/ready/rejected/purchased)
- Добавление: название, цена, категория, картинка
- 7-дневный инкубационный период перед принятием решения
- Перевод стоимости в рабочие часы

### 5. Профиль

- Имя, email, часовая ставка, язык, валюта, тема (светлая/тёмная)
- Ссылки на: Категории, Счета, Цели, Стоимость жизни, Аналитика, Выход

### 6. Категории

- Системные (userId = null) и пользовательские (userId = user.id)
- Поля: имя, тип (INCOME/EXPENSE), иконка, цвет, месячный лимит, isBaseNeed, excludeFromTotal
- Создание, редактирование, удаление
- Графики: круговая диаграмма расходов по категориям за период

### 7. Счета (Accounts)

- Типы: CASH, BANK, CREDIT, INVESTMENT, DEBT
- Баланс в копейках (BigInt), валюта
- isDefault, includeInTotal
- CRUD операции

### 8. Цели (Goals)

- Название, целевая сумма, текущая сумма, дедлайн
- Пополнения (GoalContribution: сумма, заметка, дата)
- Отметка завершения

### 9. Стоимость жизни (Life Cost)

- Калькулятор: «Эта покупка стоит X часов твоей жизни»
- Расчёт на основе часовой ставки пользователя (hourlyRate)
- Симуляция инвестиций: «Если вложить X ₽ на N лет под 12%...»

### 10. Аналитика

- Графики: расходы/доходы по категориям за период
- Тренды: линейный график по месяцам
- Сравнение периодов

### 11. Статьи (Articles)

- Бэкенд: Article + ArticleTranslation (5 языков)
- GET /api/articles?lang=ru — список с переводами
- GET /api/articles/:id — просмотр, +1 к viewCount
- Экран чтения: заголовок, контент, бейдж языка, время чтения, счётчик просмотров

### 12. Авторизация

- JWT: access (15 мин) + refresh (30 дней, ротация)
- Регистрация: email + пароль + имя + часовая ставка (опционально)
- Логин: email + пароль
- Демо-режим: без регистрации, фейковые данные
- Хранение: SecureStore (mobile), httpOnly cookie (web)

## Модели данных (Prisma)

| Модель | Назначение |
|--------|-----------|
| User | Пользователь: email, пароль, имя, валюта, язык, hourlyRate, monthlyHours |
| Session | Сессия: userId, token, expiresAt |
| Account | Счёт: name, type (CASH/BANK/CREDIT/INVESTMENT/DEBT), balance, currency |
| Category | Категория: name, type, icon, color, monthlyLimit, isBaseNeed, userId (null = системная) |
| Transaction | Транзакция: amount (BigInt копейки), type (INCOME/EXPENSE/TRANSFER), date, description |
| Goal + GoalContribution | Финансовая цель с пополнениями |
| WishlistItem | Желание: name, price, status (PENDING/READY/REJECTED/PURCHASED), cooldownDays |
| ChatMessage | Сообщение чата: role (USER/ASSISTANT), content, presetType |
| Article + ArticleTranslation | Статья с переводами на 5 языков |
| ExchangeRate | Курсы валют: code, rate, type (FIAT/CRYPTO/METAL) |
| Deposit, Loan, SavingsGoal, ForecastScenario | Депозиты, кредиты, цели накоплений, прогнозы |
| Family + FamilyMember | Семейный бюджет: роли OWNER/ADMIN/MEMBER |
| Notification | Уведомления: type, title, body, isRead |
| Gamification модели | UserGamification, Achievement, UserAchievement, XpEvent, StreakDay, Challenge, UserChallenge — **ВЫРЕЗАНЫ из UI, но есть в схеме** |

## Текущая интеграция AI

- **AiService** (backend/src/ai/) — DeepSeek через OpenAI SDK (baseURL: api.deepseek.com)
  - `parseVoiceTransaction(userId, text, language)` — парсинг текста в транзакцию
  - `parseReceiptTransaction(userId, imageBase64, mimeType, language)` — парсинг фото чека
  - Системный промпт содержит список категорий пользователя
  - Возвращает JSON: type, amount, description, categoryName, categoryType, date?

- **ChatService** (backend/src/chat/) — mock-ответы вместо DeepSeek
  - Пресеты: SPENDING_REPORT, BUDGET_ANALYSIS, SAVINGS_TIPS, DYNAMICS
  - Возвращает заготовленный текст без AI
