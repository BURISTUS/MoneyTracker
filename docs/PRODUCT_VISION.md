# Product Vision: Money Tracker

## 1. Описание продукта

**Название:** Money Tracker  
**Тип:** Мобильное приложение для личного финансового учета  
**Стек:** React Native (Expo) + NestJS + PostgreSQL + Redis  
**Лозунг:** "Трать осознанно - копи с умом"

---

## 2. Позиционирование

### Проблема на рынке
- Люди не понимают куда уходят деньги
- Используют 5+ разных приложений для финансов
- Банковские приложения не дают полной картины
- Excel слишком сложный для большинства
- Все приложения одинаковые - просто списки трат

### Наше решение
Единое приложение для полного контроля личных финансов с акцентом на **осознанное потребление** через визуализацию "стоимости жизни".

### Почему мы лучше
1. **Life Cost** - уникальная фишка, которой нет нигде
2. **Всё в одном** - не нужно 10 приложений
3. **Простой UI** - не как Excel, приятный и понятный
4. **Офлайн-first** - работает без интернета

## 3. Целевая аудитория

### Primary (Основная)
- **Возраст:** 25-45 лет
- **Доход:** средний и выше среднего (50-150к руб/мес)
- **Поведение:** Пользуется смартфоном ежедневно, любит технологии
- **Боли:** Не понимает куда уходят деньги, хочет копить но не получается
- **Мотивация:** Финансовая свобода, достижение целей

### Secondary (Вторичная)
- **Студенты (18-25 лет)** - учатся обращаться с деньгами, ограниченный бюджет
- **Молодые семьи (25-35 лет)** - совместный учет, планирование крупных покупок

## 4. Ключевая ценность (USP)

### Главная фишка: Life Cost (Стоимость жизни)

> Показывает сколько часов жизни нужно работать чтобы купить вещь.

**Как это работает:**
1. Пользователь устанавливает свою почасовую ставку (или она рассчитывается из зарплаты)
2. Каждая транзакция показывает сколько часов "жизни" она стоит
3. Визуальные подсказки: "Этот iPhone стоит 200 часов твоей жизни"

**Примеры:**
- "Этот iPhone стоит 200 часов твоей жизни"
- "Ужин в ресторане = 8 часов работы"  
- "Новая куртка = 40 часов"
- "Кофе каждый день = 2 месяца в году"

**Эффект:** Эмоциональное воздействие заставляет задуматься перед покупкой. Это не просто "потрачено 1000 руб" - это "потрачено 5 часов жизни".

**Почему это работает:**
- Люди не чувствуют деньги абстрактно
- Время - самый ценный ресурс
- Простая формула: Деньги / Почасовая ставка = Часы жизни

## 5. Инкубатор покупок (Wishlist 2.0)

### Что это
Вишлист с умом - не просто список желаний, а инструмент осознанных покупок.

### Как работает

**Добавление товара:**
1. Название товара
2. Цена
3. Приоритет (must-have / хочу / подожду)
4. Ссылка (опционально)
5. Фото (опционально)

**Аналитика:**
- "Этот товар стоит X часов жизни"
- Сколько дней экономишь на эту покупку
- История снижения/повышения цены (в будущем)

**Автоматизация:**
- Можно настроить "откладывать N руб/день" на желаемую покупку
- Автоматическое списание с определенного счета
- Уведомление когда накопилась нужная сумма

### Примеры использования

**Сценарий 1: Смартфон**
- Пользователь добавляет iPhone 15 за 80,000 руб
- При почасовой ставке 500 руб = 160 часов жизни
- При откладывании 500 руб/день = 160 дней (5 месяцев)
- Приложение показывает: "Накопишь за 5 месяцев"

**Сценарий 2: Отпуск**
- Пользователь добавляет отпуск за 150,000 руб
- При откладывании 1000 руб/день = 150 дней
- Виджет на главном экране показывает прогресс

## 6. Core Функции

### 6.1 Транзакции
- Добавление доходов и расходов
- Категоризация (продукты, транспорт, развлечения, здоровье...)
- Теги и заметки
- Регулярные платежи (подписки: Netflix, Spotify, ЖКХ)
- Быстрое добавление (3 тапа максимум)

### 6.2 Счета
- Несколько счетов (наличные, карта, вклад, инвестиции)
- Переводы между счетами
- Отслеживание общего баланса
- Валюты (руб, доллар, евро)

### 6.3 Категории
- Системные категории (preset)
- Свои категории
- Иконки и цвета
- Бюджеты по категориям

### 6.4 Бюджеты
- Месячные бюджеты по категориям
- Общий месячный бюджет
- Уведомления при превышении (70%, 90%, 100%)
- Перенос остатка на следующий месяц

### 6.5 Цели накоплений (Goals)
- Накопление на цель с дедлайном
- Автоматические отчисления
- Несколько целей одновременно
- Достижение цели - уведомление и празднование

### 6.6 Инкубатор покупок (Wishlist)
- Список желаемых покупок
- Приоритеты (must-have / хочу / подожду)
- Расчет времени накопления
- Автоотчисления

### 6.7 Life Cost (ключевая фишка)
- Настройка почасовой ставки
- Ручная или авто (из зарплаты)
- Визуализация "часов жизни" на каждой транзакции
- Статистика: сколько часов потрачено в этом месяце

### 6.8 Геймификация (переосмысленная)
- **НЕ:** баллы XP за каждую покупку
- **ДА:**
  - Сравнение с прошлым месяцем ("Потратил на 10% меньше!")
  - Достижение целей накоплений
  - Недельный челлендж ("Не превысил бюджет за неделю")
  - Streak: сколько дней подряд записываешь траты

## 7. User Flow

### Первый запуск
1. Регистрация (email + пароль)
2. Онбординг (3 экрана):
   - "Твой доход в час"
   - "Добавь свои счета"
   - "Начни добавлять траты"
3. Главный экран

### Добавление транзакции
1. Кнопка "+" на главном экране
2. Выбор типа (доход/расход)
3. Сумма (цифровая клавиатура)
4. Категория (сетка иконок)
5. Счет (выпадающий список)
6. Готово!

### Накопление на цель
1. Вишлист -> Добавить товар
2. Ввести название, цену, приоритет
3. Выбрать "Автоотчисления"
4. Выбрать счет и сумму
5. Приложение само откладывает

## 8. UI/UX Принципы

### Дизайн-система
- **Primary:** #1E3A5F (темно-синий) - доверие, стабильность
- **Secondary:** #2DD4BF (бирюзовый) - инновации, рост
- **Success:** #059669 (зеленый) - накопления, рост
- **Danger:** #DC2626 (красный) - расходы, предупреждения

### Стиль
- Минимализм
- Много белого пространства
- Карточки с закруглениями (border-radius: 16-24)
- Плавные анимации (Reanimated)
- Темная тема

### Принципы
1. **Главный экран** = баланс + последние 5 операций
2. **Добавление транзакции** = 3 тапа максимум
3. **Любой экран** = максимум 2 уровня вложенности
4. **Акцент** - на Life Cost и визуализацию

## 9. Техническая архитектура

### Frontend
- **Expo SDK 52+** - React Native
- **Zustand** - стейт-менеджмент (простой и легкий)
- **React Navigation** - навигация
- **Reanimated** - анимации

- **PostgreSQL** - основная БД
