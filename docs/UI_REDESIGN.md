# Экспертиза современных финансовых приложений и переделки дизайна

## Анализ современных финтех-приложений

### 1. Monobank
**Ключевые паттерны:**
- **Главный экран**: баланс занимает 30-40% экрана (крупный шрифт), быстрые действия в виде карточек (Переводы, Квитанции, Промокоды)
- **Создание операции**: пицца/круглый интерфейс для ввода суммы, категории через чипы
- **Навигация**: таббар с 5 вкладками, иконки крупные
- **Анимации**: плавные переходы, пружинные эффекты

### 2. Revolut
**Ключевые паттерны:**
- **Главный экран**: баланс в центре с плавающей кнопкой "+" (FAB), круговой график расходов
- **Транзакции**: список с крупными иконками категорий, дата справа, сумма - отдельным блоком
- **Выбор типа**: через bottom sheet с двумя большими кнопками (зеленый/красный)
- **Категории**: не больше 6-8 на экране, с иконками и субтитрами

### 3. Tinkoff
**Ключевые паттерны:**
- **Главный экран**: карточки со счетами в начале, быстрые действия под каждой карточкой
- **Навигация**: таббар с крупными иконками + кнопки для быстрых действий
- **Создание операции**: калькуляторный интерфейс для суммы, категории в виде сетки

### 4. 1money (Важно для этого проекта)
**Ключевые паттерны:**
- **Главный экран**: баланс в центре, **виджеты** для быстрых операций прямо на главном экране
- **Виджеты**: "Добавить расход", "Добавить доход" - большие кнопки с иконками
- **Транзакции**: список с цветовой кодировкой (красный/зеленый), крупные иконки
- **Выбор суммы**: большие круглые кнопки (100, 500, 1000, 5000, 10000) или свайп-селектор
- **Категории**: максимум 6-8 категорий на экране, с emoji-иконками

### 5. Raiffeisen
**Ключевые паттерны:**
- **Главный экран**: баланс в центре, две большие кнопки внизу "Перевести" и "История"
- **Минимализм**: максимум 1-2 действия на экране
- **Создание операции**: пицца для выбора суммы + категории на следующем экране

## Общие паттерны успешных финтех-приложений

### 1. Главный экран
✅ **Должно быть:**
- Баланс занимает 25-35% экрана (очень крупный шрифт)
- Максимум 3-4 быстрых действия в виде крупных кнопок/карточек
- Список транзакций занимает оставшееся место (максимум 5-7 операций)
- FAB (плавающая кнопка) для быстрого добавления

❌ **Не должно быть:**
- Сложные графики/диаграммы на главном экране
- Много мелких элементов
- Больше 4 быстрых действий
- Меню настроек в центре внимания

### 2. Создание транзакции
✅ **Должно быть:**
- **Первый экран**: выбор типа (доход/расход) + ввод суммы через калькулятор/пиццу
- Быстрые значения суммы: 100, 500, 1000, 5000, 10000 в виде больших кнопок
- **Категории на этом же экране** (НЕ отдельный шаг)
- Максимум 6-8 категорий на экране
- Одна большая кнопка "Сохранить" внизу

❌ **Не должно быть:**
- Много полей (счет, описание)
- Отдельный экран для выбора категории
- Маленькие чипы для категорий (должны быть большими кнопками)
- Сложные анимации переходов

### 3. Навигация
✅ **Должно быть:**
- Таббар снизу с 4-5 вкладками
- Крупные иконки (минимум 32px)
- Подписи под иконками
- Активная вкладка ярко выделена

❌ **Не должно быть:**
- Больше 5 вкладок в таббаре
- Мелкие иконки
- Многоуровневое меню

### 4. Категории
✅ **Должно быть:**
- Иконки + названия (emoji или иконки)
- Крупные кнопки (минимум 60px высота)
- Максимум 6-8 категорий на экране
- Цветовая кодировка (доход - зеленый, расход - красный)
- Остальные категории скрыты за кнопку "Ещё"

❌ **Не должно быть:**
- Только названия без иконок
- Мелкие чипы (менее 48px высота)
- Больше 10 категорий на экране
- Нет визуального различия между типами категорий

## Общие принципы переделки

### 1. Минимализм
- Убрать всё, что не нужно для основных действий
- Максимум 3-4 элемента на экране
- Один главный акцент на каждом экране

### 2. Крупные элементы
- Кнопки минимум 56px высота
- Чипы категорий минимум 60px высота
- Шрифт баланса минимум 60px

### 3. Цветовая кодировка
- Доход = зеленый (#34D399)
- Расход = красный (#F87171)
- Остальное = серый/нейтральный

### 4. Меньше шагов
- Создание транзакции = 1 экран (не 3-4)
- Минимум кликов для выполнения действий

### 5. Виджеты на главном экране
- Быстрые действия прямо на главном экране
- Крупные кнопки с иконками
- Не скрывать за меню

### 6. Категории
- Максимум 6-8 категорий на экране
- Emoji-иконки или крупные иконки
- "Ещё" для остальных

## Технические требования

### Размеры компонентов
- Кнопка: минимум 56px высота, 16px паддинги
- Чип категории: минимум 60px высота, 20px паддинги
- Баланс: минимум 60px шрифт
- Карточка транзакции: минимум 72px высота

### Цвета
- Primary: #6366F1
- Success (доход): #34D399
- Danger (расход): #F87171
- Background: #0A0A0F
- Card: rgba(255, 255, 255, 0.04)

### Типографика
- Баланс: xxxl (60px+)
- Заголовки: xl (24px)
- Тело: md (16px)
- Подписи: sm (14px)

### Анимации
- Пружинные (spring) для нажатий
- Fade для переходов между экранами
- Slide для навигации (не все одновременно)

## Заключение

Главный принцип: **"Меньше элементов, крупнее кнопки, больше действий на главном экране"**

Пользователь должен уметь:
1. Увидеть баланс за 0.5 секунды
2. Добавить расход за 1-2 клика
3. Найти нужное за 1-2 клика
4. Не думать над интерфейсом, использовать интуитивно
