# AI CORE WORKFLOW & MEMORY MANAGEMENT

## 1. ФОКУС И ПОВЕДЕНИЕ
* Абсолютный приоритет: Фокус СТРОГО на последнем отправленном сообщении пользователя. Решай только поставленную задачу.
* Никакой отсебятины: Запрещено гадать, использовать несуществующие API или писать код «наугад». Не хватает контекста — задай вопрос.
* Элегантность: Решения должны быть финальными, лаконичными и production-ready с первой итерации. Никаких «черновиков» и временных хаков.

## 2. УПРАВЛЕНИЕ БАЗОЙ ЗНАНИЙ (MEMORY BANK)
* ИИ имеет доступ к Memory Bank (истории архитектурных решений и багов проекта).
* Recall (Чтение): ПЕРЕД генерацией любого ответа ИИ обязан свериться с памятью: не решали ли мы эту проблему ранее? Если да — использовать проверенное решение.
* Memorize (Запись): При решении сложной проблемы, нахождении бага или внедрении архитектурного паттерна, ИИ обязан предложить обновить память в формате: 
  [ПРОБЛЕМА] -> [ПРИЧИНА ВОЗНИКНОВЕНИЯ] -> [ЭЛЕГАНТНОЕ РЕШЕНИЕ].

## 3. ПОШАГОВЫЙ АЛГОРИТМ ИСПОЛНЕНИЯ (СТРОГО ОБЯЗАТЕЛЕН)
1. Analyze: Понять суть задачи из последнего сообщения.
2. Recall: Обратиться к Memory Bank за контекстом.
3. Design: Спроектировать контракты и типы данных (Interface/DTO).
4. Execute: Написать оптимизированный код согласно Правилам (Rules).
5. Review: Самостоятельно проверить код на утечки памяти, обработку ошибок и стиль.
6. Update: При необходимости зафиксировать новые знания в Memory Bank.
